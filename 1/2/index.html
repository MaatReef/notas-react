4. Entendiendo la carpeta public

        --------------------------------------------------------

        - public		-> Donde inicia el proyecto
                    -  Lo que se cargará al navegador web

                -> < html lang="es">	< html lang="en">
                - Carga los favicon
                - el manifest.json	-> Archivo importante para PWA, aplicación web progresiva.
                
                %PUBLIC_URL%		-> Todo lo que llamemos de este html debe estar dentro de la carpeta public.					

        
        < body>
            < noscript> You need to enable Javascript to run this app 	-> Cuando un navegador no encuentra js, renderiza este mensaje.

                - Todos los metas o analytics, plugins se pueden instalar dentro del archivo index.. SIn incovenientes.

5. El componente index


        --------------------------------------------------------
        *Terminal desde la consola de Visual Studio COde o bien desde la Terminal como tal*
        
            -> yarn start			-> Inicamos la aplicación.
        
        
        --------------------------------------------------------
            - Carpeta 	-> 	src

        
        index.js
            -> Archivo inicial donde comiena nuestra aplicación y nuestros archivos se renderizan.
               SOLO en el index.js es necesario importar React y ReactDom
        
        import React from 'react';
        import ReactDom from 'react-dom';
        import './index.css';					-> Estilos css
        import App from './App';				-> El arcivo app.js
        
        ReactDom.render(
            < React.StrictMode>		
                < App />					-> UN HIJO (un elemento, con varios dentro)
            ,
            document.getElementById('root')			-> Div del index.html de la carpeta public, a ese div le inyectamos hijos.
        );
                                                            *No es un requisito pero es útil*
                                                        -> Evuelve a toda la aplicación que creamos.
                    -> < React.StrictMode> Traemos desde React el componente .StrictMode	-> Permite visualizar advertencias, errores..	
                                                            - Mal renderizado de componentes
                                                            - Mal llamado a otro tipos de componentes
                                                            - Cambios de estados de mala manera, etc	
            
        
        --------------------------------------------------------
                Importamos de manera destructura StrictMode
        
        
                                    * 'react' es útil para renderizar el componente StrictMode, si no lo necesitamos lo quitamos.
        import { StrictMode } from 'react';		-> Desde aca, de la librería React
        import { render } from 'react-dom';		-> Lo mismo con render
        import './index.css';					
        import App from './App';				
        
        render(
            < StrictMode>		
                < App />					
            ,
            document.getElementById('root')			
        );
        
        
            -> Por lo tanto tanto como 'react' y 'react-dom', nos ayudan a importar StrictMode y render, para utilizarlo.
        
        *Todo esto es la puerta de Acceso a la Aplicación*
                - La aplicación es un componente 	-> < App />
        
                import App from './App';	-> Ese el archivo el cual llamamos.
